# ゲーム仕様書：『Viral Ranker（仮）』

## 1. ゲームタイトル
**Viral Ranker（バイラル・ランカー）**
※SNSでの拡散を前提とした、シンプルかつ中毒性の高いスコアアタックゲーム。

## 2. 概要
本作は、短時間でプレイ可能なスコアアタックゲームです。最大の特徴は、**「プレイ終了後にSNS投稿用の最適化された画像（スコアカード）が自動生成される」**点と、**「リアルタイム全国ランキング」**を実装している点です。プレイヤーの承認欲求を刺激し、UGC（ユーザー生成コンテンツ）を自然発生させることで、広告費をかけずにユーザーベースを拡大する戦略をとります。

## 3. コア体験 / ゲーム性
*   **コア体験:** 「自己ベスト更新の喜び」と「SNSでの自慢・共有」のサイクル。
*   **ゲーム性:**
    *   1プレイ60秒の超高速アクション。
    *   スコアに応じて「ランク（S〜D）」や「称号」が付与される。
    *   **SNS連動:** 終了画面で「シェアボタン」を押すと、スコア・順位・称号が記載された「映える画像」が自動生成され、X（旧Twitter）等のアプリへ直接遷移・投稿可能。
    *   **ランキング:** 週間/月間/累計のランキングを実装し、上位者には特別なデジタルバッジを付与。

## 4. 操作方法
*   **基本操作:** タップ、長押し、スワイプのみの直感的なワンハンド操作。
*   **UI:** 画面下部に常に「シェア」ボタンを配置し、プレイ終了後3秒以内にSNS投稿画面へ移行できる導線を確保。

## 5. 開発手順（エンジニアAIへの指示用）

エンジニアAIに対し、以下のステップで実装を依頼してください。

### Step 1: ゲームエンジンの選定とプロトタイプ作成
*   **指示:** 「UnityまたはGodotを使用し、60秒間のスコアアタックの基本ループを作成せよ。スコア計算ロジックと、ゲームオーバー時のリザルト画面UIを実装すること。」

### Step 2: 画像生成エンジンの実装
*   **指示:** 「リザルト画面の情報を元に、動的に画像（PNG形式）を生成するモジュールを作成せよ。背景画像にスコア、順位、称号をオーバーレイ合成し、端末のローカルストレージに保存する機能を実装すること。」

### Step 3: SNS連携機能の実装
*   **指示:** 「OS標準の共有機能（Native Share）を実装し、生成された画像をSNSアプリへ直接渡せるようにせよ。投稿文にはハッシュタグ `#ViralRanker` と現在のスコアが自動挿入されるようにすること。」

### Step 4: バックエンド・ランキング機能の実装
*   **指示:** 「Firebase (Firestore/Authentication) を使用し、ユーザーのスコアをリアルタイムで集計するランキングシステムを構築せよ。ランキングは『週間』と『全期間』の2種類を用意し、API経由で取得できるようにすること。」

### Step 5: UI/UXの磨き込みとSNS最適化
*   **指示:** 「SNSでシェアされた際に最も見栄えが良くなるよう、リザルト画像のレイアウトを調整せよ。また、ランキング上位者がSNSでシェアしたくなるような『限定称号』の表示ロジックを追加すること。」
