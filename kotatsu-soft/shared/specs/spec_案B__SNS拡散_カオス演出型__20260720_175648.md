# ゲーム仕様書：『Chaos Physics Playground（仮）』

## 1. ゲームタイトル
**『Chaos Physics Playground』**
（サブタイトル案：宇宙で暴れる物理演算の宴）

## 2. 概要
本作は、プレイヤーがオブジェクトを配置し、物理演算によって引き起こされる「予測不能なカオス」を楽しむサンドボックス型ゲームです。背景が動的に切り替わるシュールな環境と、物理法則を無視したような過剰な挙動により、SNSでの「切り抜き動画」による拡散を最大化することを目的とします。

## 3. コア体験 / ゲーム性
*   **カオス・サンドボックス:** プレイヤーは様々な形状のオブジェクトを配置し、それらを衝突・爆発・結合させます。
*   **環境の動的変化:** 一定時間ごと、または特定のトリガーで背景が「深海」「宇宙空間」「溶岩地帯」「無重力オフィス」など、全く異なるシュールな環境へシームレスに切り替わります。
*   **過剰な物理演算:** 衝突時の反発係数を極端に高く設定し、オブジェクトが画面外へ弾け飛んだり、重なり合って爆発的な挙動を見せる「映える」瞬間を演出します。
*   **SNSシェア機能:** 直近30秒のプレイを自動録画し、ワンボタンでSNS投稿用フォーマット（縦型動画）に書き出す機能を実装します。

## 4. 操作方法
*   **マウス左クリック:** オブジェクトの配置 / 掴んで投げる
*   **マウス右クリック:** オブジェクトの削除 / 物理演算の強制リセット
*   **マウスホイール:** オブジェクトのサイズ変更
*   **スペースキー:** 環境の強制切り替え（クールタイムあり）
*   **Rキー:** 録画開始/停止（SNSシェアメニュー呼び出し）

## 5. 開発手順（エンジニアAIへの指示用）

### フェーズ1：物理演算基盤の構築
*   Unity/Godot等の物理エンジンを使用し、オブジェクト同士の衝突判定と反発係数の調整を行う。
*   「過剰な挙動」を実現するため、`Rigidbody`の`Drag`や`Angular Drag`を動的に変更するスクリプトを作成すること。

### フェーズ2：環境切り替えシステムの構築
*   背景（Skybox/背景画像）と物理環境（重力値、空気抵抗）をセットにした「環境プリセット」を定義する。
*   `EnvironmentManager`クラスを作成し、タイマーまたはトリガーに応じて環境を切り替える機能を実装する。

### フェーズ3：カオス演出の強化
*   オブジェクトが一定速度を超えた際に、パーティクル（火花、煙、残像）が発生するエフェクトを追加する。
*   画面揺れ（Screen Shake）を物理演算の衝撃量に連動させる。

### フェーズ4：SNS拡散用機能の実装
*   Unity Recorder等のライブラリを活用し、バックグラウンドで録画を行う機能を実装する。
*   UI上に「シェアボタン」を配置し、動画ファイルをローカルに保存、またはクリップボードにコピーする処理を実装する。
