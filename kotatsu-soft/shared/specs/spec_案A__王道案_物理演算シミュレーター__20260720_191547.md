# 仕様書：物理演算積み上げシミュレーター（仮題：Stack Physics Pro）

## 1. ゲームタイトル
**Stack Physics Pro**

## 2. 概要
本作は、物理演算エンジンを活用した「オブジェクト積み上げ」シミュレーターです。プレイヤーは様々な形状のオブジェクトをバランスよく積み上げ、崩さずにどこまで高く積み上げられるかを競います。物理挙動のリアリティを追求し、摩擦係数や重心位置をJSONで一元管理することで、拡張性と難易度調整の容易さを両立させます。

## 3. コア体験 / ゲーム性
*   **コア体験:** 「重力と摩擦の限界に挑む」緊張感と、完璧なバランスで積み上がった際の達成感。
*   **ゲーム性:**
    *   **物理挙動:** 剛体（Rigidbody）の挙動を重視。オブジェクト同士の接触判定と摩擦が積み上げの成否を分ける。
    *   **データ駆動型設計:** 各オブジェクトの特性（質量、摩擦係数、反発係数、重心オフセット）をJSONで定義。これにより、ステージごとの難易度調整や、新しいオブジェクトの追加を容易にする。
    *   **中毒性:** 物理演算特有の「予測不能な挙動」と「精密な操作」のバランス。崩れそうで崩れない絶妙なラインを攻める楽しさを提供。

## 4. 操作方法
*   **マウス/タッチ操作:**
    *   **ドラッグ:** オブジェクトを水平方向に移動。
    *   **クリック/タップ:** オブジェクトを落下させる（または配置する）。
    *   **ホイール/ピンチ:** カメラのズームイン・アウト。
*   **UI操作:**
    *   **リセットボタン:** 現在の積み上げを破棄し、初期状態に戻す。
    *   **オブジェクト選択:** 次に配置するオブジェクトの切り替え。

## 5. 開発手順（エンジニアAIへの指示用）

### Step 1: プロジェクト基盤の構築
*   物理エンジン（Unity PhysX または Matter.js等）のセットアップ。
*   オブジェクトのプロパティを管理する`config.json`のスキーマ定義。
    *   *JSON構造例:* `{"name": "cube", "mass": 1.0, "friction": 0.5, "bounciness": 0.1, "centerOfMass": [0, 0, 0]}`

### Step 2: 物理挙動の実装
*   JSONデータを読み込み、動的に物理マテリアル（Physics Material）を生成・適用するスクリプトの実装。
*   オブジェクトの落下・衝突判定の最適化。

### Step 3: ゲームループとUIの実装
*   配置フェーズ（移動）→ 落下フェーズ（物理演算開始）のステート管理。
*   高さ計測機能（現在の最高到達点をUIに表示）。
*   崩壊判定（オブジェクトが地面に接触、または一定時間静止しない場合の処理）。

### Step 4: バランス調整とデバッグ
*   摩擦係数と重心位置を調整し、ゲームとして成立する「難易度曲線」の作成。
*   エッジケース（オブジェクトが重なった際の挙動など）の修正。
